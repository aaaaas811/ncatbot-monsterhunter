# MH 插件

怪物猎人相关功能的QQ机器人插件，支持集会码管理、怪物数据查询等功能。

## 功能特性

### 集会码管理
- **自动记录**：发送MHW或MHR集会码自动记录
- **查询列表**：查看所有记录的集会码
- **删除功能**：删除最近的集会码
- **清空功能**：清空所有集会码

### 怪物数据查询（目前仅限Wilds）
- **怪物列表**：列出所有收录的怪物名称
- **怪物简介**：查询怪物的基本信息、图片、简介等
- **弱点分析**：分析怪物的弱点，提供物理和属性攻击建议
- **肉质表**：显示怪物的肉质数据和状态分析

### 数据更新
- **自动爬取**：从网络获取最新的怪物数据
- **数据存储**：数据保存在本地，支持离线查询

## 使用方法

### 集会码相关命令
- 发送12位MHW集会码或8位MHR集会码自动记录
- `/查询` - 查看所有集会码列表
- `/删除mhw` - 删除最近的MHW集会码
- `/删除mhr` - 删除最近的MHR集会码
- `/清空` - 清空所有集会码

### 怪物查询命令（仅限Wilds）
- `/怪物列表` - 显示所有怪物名称
- `/简介 [怪物名字]` - 查询怪物基本信息
- `/弱点 [怪物名字]` - 查看怪物弱点分析
- `/肉质 [怪物名字]` - 显示怪物肉质表

### 管理命令
- `/helpMH` - 显示帮助信息
- `/爬取ws` - 更新怪物数据（需要网络连接）

## 安装依赖

插件需要以下Python包，请在插件目录下运行：

```bash
pip install -r requirements.txt
```

依赖包列表：
- requests==2.31.0
- beautifulsoup4==4.12.2
- aiohttp>=3.9.1

## 数据存储

- 集会码数据存储在内存中，重启后丢失
- 怪物数据存储在 `data/` 文件夹中
- 使用 `.gitignore` 忽略数据文件夹，避免提交到版本控制

## 注意事项

1. 怪物数据查询功能目前仅支持怪物猎人Wilds
2. 爬取数据功能需要网络连接，可能需要较长时间
3. 初始化以及版本更新时，建议运行 `/爬取ws` 命令更新数据
4. 集会码格式会自动验证，无效格式不会记录

## 技术架构

- 主文件：`mh.py` - 插件入口和消息处理
- 分析模块：`analyze.py` - 怪物数据分析逻辑
- 爬虫模块：`mhws_Wiki_Crawler/` - 数据爬取工具
- 数据存储：`data/` - JSON格式的怪物数据
